Portainer is a web UI for managing Docker on your home servers. This dashboard uses its API to detect when your container images have updates available and to trigger redeployments. You only need this if you run Docker containers.

How to find these values in Portainer
  1. Open Portainer in your browser (usually https://your-server-ip:9443)
  2. Click your username in the top-right corner → Account Settings
  3. Scroll down to Access Tokens → click Add access token
  4. Give it a name like update-dashboard and click Generate
  5. Copy the token — you won't be able to see it again

Include https:// and the port number (usually 9443)

Docker Hub Optional
{% if conn.dockerhub_user and conn.dockerhub_token_set %} Configured {% else %} Not set up {% endif %}

When this dashboard checks if your container images have newer versions available, it asks Docker Hub. Without an account, Docker Hub limits this to 100 checks per hour shared across everyone on your IP address. With a free account you get 200 checks per hour just for you. If you run more than a handful of containers, you'll want this set up.

How to create a Docker Hub access token
  1. Sign in at hub.docker.com (free account is enough)
  2. Click your username (top-right) → Account Settings
  3. Go to Personal access tokens → Generate new token
  4. Give it a name like update-dashboard, set access to Public Repo Read-only
  5. Copy the token — you won't see it again

Use an access token, not your Docker Hub password. Tokens are scoped and can be revoked individually if they ever leak.
