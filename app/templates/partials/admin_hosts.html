How host connections work

The dashboard connects to each host over SSH to run package checks and upgrades. Choose an auth method and make sure the remote host accepts it.

SSH Key (recommended)

  1. Generate a dedicated key pair next to your docker-compose.yml:
    ssh-keygen -t ed25519 -f keys/id_ed25519 -N ""
  2. Authorize it on each remote host:
    ssh-copy-id -i keys/id_ed25519.pub user@host
  3. The keys/ directory is already mounted into the container by default. Set the key path per-host below, or leave it blank to use the default from SSH Settings.

Password

  1. Create a dedicated user on the remote host:
    adduser dashboard
  2. Grant it sudo without a password prompt for apt:
    echo "dashboard ALL=(ALL) NOPASSWD: /usr/bin/apt*" > /etc/sudoers.d/dashboard
  3. Ensure PasswordAuthentication yes is set in /etc/ssh/sshd_config, then restart SSH:
    systemctl restart sshd

Passwords are stored in plaintext in config.yml. Always use a dedicated user — never root.
